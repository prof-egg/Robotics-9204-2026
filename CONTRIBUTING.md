# Contributing to FRC Team 9204 2026 Robot Code

> _This just contains placeholder text for now, a proper contributing file is on the todo list._

Thanks for contributing! Please follow these guidelines.

## Branching
- Always create a **feature branch**: `feature/<name>` or `bugfix/<name>`
- Never push directly to `main`

## Commits
- Use present tense: `feat: add shooter PID`
- Keep messages short and descriptive

## Pull Requests
- Open a PR to merge into `main` or `development`
- Add a description of the change
- Request review from a mentor before merging

## Code Style
- Use PascalCase for classes
- camelCase for methods and variables
- Keep methods short and readable

## Tests
- Always test your code on the practice robot or simulator before merging
